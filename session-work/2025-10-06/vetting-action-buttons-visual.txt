• Maintains 48px height for touch targets
• 16px spacing prevents mis-taps


HANDLER FUNCTIONS
=================

handleAdvanceStage()
├── Checks nextStageConfig exists
├── Builds reasoning string with label and description
└── Calls appropriate mutation:
    ├── approveApplication() if next status = 'Approved'
    └── submitDecision() for intermediate stages

handleSkipToApproved()
├── Builds "skipped to approval" reasoning
└── Calls approveApplication() mutation

handlePutOnHold()
└── Opens OnHoldModal

handleSendReminder()
└── Opens SendReminderModal

handleDenyApplication()
└── Opens DenyApplicationModal


ACCESSIBILITY FEATURES
======================

✓ Keyboard Navigation
  - Tab order: Tier 1 → Tier 2 (L→R) → Tier 3 (L→R)
  - Enter/Space activates buttons
  - Disabled buttons skip in tab order

✓ Screen Readers
  - Descriptive test IDs
  - Loading states announced
  - Disabled states clear

✓ Visual Accessibility
  - Color contrast 4.5:1 minimum (WCAG 2.1 AA)
  - Touch targets: 56px, 48px, 36px
  - Focus indicators: 2px outline + 4px offset

✓ Reduced Motion
  - Respects prefers-reduced-motion
  - No translateY movement per design system
